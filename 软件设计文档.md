## 一、引言

### 1.1 编写目的

本文档是线上点餐系统的软件设计文档，旨在详细描述系统的整体架构、功能模块、数据库设计、接口规范等内容。本文档面向开发人员、测试人员和指导教师，作为系统开发、测试和验收的依据。

### 1.2 项目背景

随着餐饮行业的数字化转型，传统纸质菜单点餐方式已无法满足现代餐厅的效率需求。本项目旨在开发一套**同时支持实体店堂食和外卖配送**的线上点餐系统，实现顾客自助点餐、外卖下单、厨房订单管理、经营数据分析等功能，提升餐厅运营效率。

本项目为C#课程大作业，由3人团队协作完成，开发周期为2周。

## 二、系统概述

### 2.1 系统目标

开发一套完整的线上点餐系统，**同时支持实体店堂食和外卖配送**两种模式，包含以下核心功能：

1. **顾客端**：浏览菜单、选择菜品、堂食/外卖切换、下单、查看订单状态
2. **管理端**：菜品管理、订单管理（区分堂食/外卖）、统计分析、AI经营建议
3. **后端服务**：提供RESTful API接口、数据存储、AI推荐服务

### 3.2 架构说明

#### 3.2.1 表示层

表示层负责与用户交互，包括两个独立的WinForms桌面应用程序：

- **顾客端**：运行在餐厅餐桌的平板或触摸屏设备上，顾客通过该界面浏览菜单、选择菜品、提交订单。
- **管理端**：运行在餐厅前台或厨房的电脑上，工作人员通过该界面管理菜品、处理订单、查看统计数据。

表示层通过HttpClient调用后端WebAPI的RESTful接口获取和提交数据。

#### 3.2.2 业务逻辑层

业务逻辑层是系统的核心，采用ASP.NET Core WebAPI实现，主要职责：

- 接收并验证客户端请求
- 执行业务逻辑处理
- 调用数据访问层进行数据操作
- 返回处理结果

包含以下组件：
- **Controllers**：接收HTTP请求，调用Service处理业务
- **Services**：封装业务逻辑，如统计服务、AI推荐服务
- **DTOs**：定义数据传输对象，规范接口数据格式

#### 3.2.3 数据访问层

数据访问层负责与数据库交互，使用Entity Framework Core实现ORM映射：

- **AppDbContext**：数据库上下文，管理实体与数据库表的映射
- **DbSet<T>**：表示数据库中的表
- **Migrations**：数据库迁移文件，管理数据库结构变更

### 4.2 数据表设计

#### 4.2.1 菜品表（Dishes）

| 字段名 | 数据类型 | 约束 | 说明 |
| ------ | -------- | ---- | ---- |
| Id | INT | PRIMARY KEY, AUTO_INCREMENT | 菜品ID |
| Name | VARCHAR(100) | NOT NULL | 菜品名称 |
| Category | VARCHAR(50) | NOT NULL | 菜品分类（如：热菜、凉菜、主食、饮品） |
| Price | DECIMAL(10,2) | NOT NULL | 菜品价格 |
| ImagePath | VARCHAR(255) | NULL | 菜品图片路径 |
| SpicyLevel | INT | DEFAULT 0 | 辣度等级（0-不辣，1-微辣，2-中辣，3-特辣） |
| IsAvailable | BOOLEAN | DEFAULT TRUE | 是否可售 |
| Description | VARCHAR(500) | NULL | 菜品描述 |

**SQL建表语句：**
```sql
CREATE TABLE Dishes (
    Id INT AUTO_INCREMENT PRIMARY KEY,
    Name VARCHAR(100) NOT NULL,
    Category VARCHAR(50) NOT NULL,
    Price DECIMAL(10,2) NOT NULL,
    ImagePath VARCHAR(255),
    SpicyLevel INT DEFAULT 0,
    IsAvailable BOOLEAN DEFAULT TRUE,
    Description VARCHAR(500)
);
```

#### 4.2.2 订单表（Orders）

| 字段名 | 数据类型 | 约束 | 说明 |
| ------ | -------- | ---- | ---- |
| Id | INT | PRIMARY KEY, AUTO_INCREMENT | 订单ID |
| CustomerName | VARCHAR(50) | NOT NULL | 顾客姓名 |
| Phone | VARCHAR(20) | NOT NULL | 联系电话 |
| Address | VARCHAR(500) | NULL | 配送地址（外卖时必填） |
| TableNumber | VARCHAR(20) | NULL | 桌号（堂食时必填） |
| Note | VARCHAR(500) | NULL | 订单备注 |
| TotalAmount | DECIMAL(10,2) | NOT NULL | 订单总金额（含配送费） |
| DeliveryFee | DECIMAL(10,2) | DEFAULT 0.00 | 配送费用 |
| OrderType | VARCHAR(20) | NOT NULL | 订单类型：DineIn（堂食）/ Delivery（外卖） |
| Status | VARCHAR(20) | DEFAULT 'Pending' | 订单状态 |
| CreatedAt | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**订单类型说明：**
- DineIn：堂食订单（顾客到店就餐）
- Delivery：外卖订单（需要配送）

**订单状态说明：**
- Pending：待处理
- Confirmed：已确认
| Preparing：制作中
| Ready：已出餐/待配送
| Delivering：配送中（仅外卖）
| Completed：已完成
| Cancelled：已取消

**SQL建表语句：**
```sql
CREATE TABLE Orders (
    Id INT AUTO_INCREMENT PRIMARY KEY,
    CustomerName VARCHAR(50) NOT NULL,
    Phone VARCHAR(20) NOT NULL,
    Address VARCHAR(500),
    TableNumber VARCHAR(20),
    Note VARCHAR(500),
    TotalAmount DECIMAL(10,2) NOT NULL,
    DeliveryFee DECIMAL(10,2) DEFAULT 0.00,
    OrderType VARCHAR(20) NOT NULL,
    Status VARCHAR(20) DEFAULT 'Pending',
    CreatedAt DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.2.3 订单明细表（OrderItems）

| 字段名 | 数据类型 | 约束 | 说明 |
| ------ | -------- | ---- | ---- |
| Id | INT | PRIMARY KEY, AUTO_INCREMENT | 明细ID |
| OrderId | INT | FOREIGN KEY | 订单ID |
| DishId | INT | FOREIGN KEY | 菜品ID |
| DishName | VARCHAR(100) | NOT NULL | 菜品名称（冗余存储） |
| Price | DECIMAL(10,2) | NOT NULL | 单价（下单时价格） |
| Quantity | INT | NOT NULL | 数量 |

**SQL建表语句：**
```sql
CREATE TABLE OrderItems (
    Id INT AUTO_INCREMENT PRIMARY KEY,
    OrderId INT NOT NULL,
    DishId INT NOT NULL,
    DishName VARCHAR(100) NOT NULL,
    Price DECIMAL(10,2) NOT NULL,
    Quantity INT NOT NULL,
    FOREIGN KEY (OrderId) REFERENCES Orders(Id) ON DELETE CASCADE,
    FOREIGN KEY (DishId) REFERENCES Dishes(Id)
);
```

### 4.4 数据库上下文

```csharp
using Microsoft.EntityFrameworkCore;
using OnlineOrdering.API.Models;

namespace OnlineOrdering.API.Data
{
    public class AppDbContext : DbContext
    {
        public AppDbContext(DbContextOptions<AppDbContext> options) : base(options) { }

        public DbSet<Dish> Dishes { get; set; }
        public DbSet<Order> Orders { get; set; }
        public DbSet<OrderItem> OrderItems { get; set; }

        protected override void OnModelCreating(ModelBuilder modelBuilder)
        {
            modelBuilder.Entity<OrderItem>()
                .HasOne(oi => oi.Order)
                .WithMany(o => o.OrderItems)
                .HasForeignKey(oi => oi.OrderId);
        }
    }
}
```

```csharp
public static class ApiHelper
{
    private static readonly HttpClient _client;
    
    static ApiHelper()
    {
        _client = new HttpClient
        {
            BaseAddress = new Uri("http://localhost:5000/api/")
        };
    }
    
    public static async Task<List<DishDto>> GetDishesAsync()
    {
        var response = await _client.GetAsync("dishes");
        response.EnsureSuccessStatusCode();
        var json = await response.Content.ReadAsStringAsync();
        return JsonConvert.DeserializeObject<List<DishDto>>(json);
    }
    
    public static async Task<int> CreateOrderAsync(OrderCreateDto order)
    {
        var json = JsonConvert.SerializeObject(order);
        var content = new StringContent(json, Encoding.UTF8, "application/json");
        var response = await _client.PostAsync("orders", content);
        response.EnsureSuccessStatusCode();
        var result = await response.Content.ReadAsStringAsync();
        return JsonConvert.DeserializeObject<OrderDto>(result).Id;
    }
}
```

## 七、技术选型说明

### 7.1 技术栈总览

| 层次 | 技术选型 | 版本 | 选型理由 |
| ---- | -------- | ---- | -------- |
| 后端框架 | ASP.NET Core WebAPI | 9.0 | 微软官方框架，与C#无缝集成，性能优秀 |
| ORM框架 | Entity Framework Core | 9.0 | 微软官方ORM，简化数据库操作 |
| 数据库 | MySQL | 8.0 | 开源免费，社区活跃，适合中小型项目 |
| MySQL驱动 | Pomelo.EntityFrameworkCore.MySql | 9.0 | EF Core官方推荐的MySQL驱动 |
| 桌面框架 | Windows Forms | .NET 9.0 | 微软桌面开发框架，学习曲线平缓 |
| JSON处理 | Newtonsoft.Json | 13.x | 流行的JSON库，序列化功能强大 |
| API文档 | Swagger/Swashbuckle | 6.x | 自动生成API文档，便于测试 |
| AI服务 | SiliconFlow/阿里云百炼 | - | 提供大语言模型API |

### 7.2 为什么选择这些技术

#### 7.2.1 ASP.NET Core WebAPI

**优点：**
- 跨平台：可在Windows、Linux、macOS上运行
- 高性能：ASP.NET Core是目前性能最高的Web框架之一
- 依赖注入：内置DI容器，便于实现松耦合
- 中间件管道：灵活的请求处理管道
- Swagger集成：自动生成API文档

**适用场景：**
- RESTful API服务
- 微服务架构
- 企业级Web应用

#### 7.2.2 Entity Framework Core

**优点：**
- LINQ查询：使用C#代码查询数据库，无需手写SQL
- 自动迁移：管理数据库结构变更
- 变更追踪：自动追踪实体状态变化

## 八、安全设计

### 8.1 数据库安全

#### 8.1.1 连接字符串保护

**不要将密码硬编码在代码中！**

```csharp
// 错误做法
"Server=localhost;Database=OnlineOrdering;User=root;Password=123456;"

// 正确做法：使用环境变量或配置文件
var connectionString = builder.Configuration.GetConnectionString("DefaultConnection");
```

#### 8.1.2 参数化查询

EF Core自动使用参数化查询，防止SQL注入：

```csharp
// EF Core会自动生成参数化SQL
var dish = await _context.Dishes.FirstOrDefaultAsync(d => d.Name == name);
```

### 8.2 API安全

#### 8.2.1 CORS配置

```csharp
builder.Services.AddCors(options =>
{
    options.AddPolicy("AllowAll", policy =>
    {
        policy.AllowAnyOrigin()
```

```csharp
policy.WithOrigins("http://localhost:3000", "http://example.com")
```

```csharp
public class GlobalExceptionHandler : IExceptionHandler

        var response = new
```

### 11.3 常见问题解答

**Q1: 数据库连接失败怎么办？**

A: 检查以下几点：
1. MySQL服务是否启动
2. 连接字符串是否正确
3. 用户名密码是否正确
4. 防火墙是否开放3306端口

**Q2: WinForms界面卡顿怎么办？**

A: 使用异步编程和Invoke更新UI：
```csharp
private async void BtnLoad_Click(object sender, EventArgs e)
{
    var data = await ApiHelper.GetDishesAsync();
    this.Invoke((MethodInvoker)delegate
    {
        dataGridView1.DataSource = data;
    });
}
```

**Q3: API返回500错误怎么办？**

A: 检查以下几点：
1. 查看控制台错误日志
2. 检查数据库连接
3. 检查请求参数格式
4. 使用Swagger测试API
